44 Building
45 --------
46 
47 It's important to note that the platform under which this project is built plays a huge role on the usability of the library.
48 
49 When compiling with the x86 or x64 platform set in build configuration, Colore will **only work on the respective system platform**
50 (32-bit if compiled using x86, and 64-bit if compiled using x64).
51 
52 The native methods are imported using `DllImport` when Colore is compiled in x86 or x64, which is why the setting matters for deployment,
53 as this cannot be changed at runtime.
54 
55 However, if compiling with the "Any CPU" configuration, Colore will dynamically load functions relevant for the current executing platform,
56 making it run on both 32- and 64-bit systems without any work having to be done by the dev.
57 
58 For non-performance critical applications, the "Any CPU" mode should be fine (this is also what the NuGet package is compiled against).
59 
60 For applications that require peak performance, we recommend shipping separate 32- and 64-bit builds of your application, using the relevant build configurations in Colore.
61 
62 The below example compiles Colore in Release mode for the x86 (32-bit) platform.
63 
64 ```
65 msbuild Colore.sln /p:Configuration=Release;Platform=x86
66 ```
67 
68 (Replace `x86` with `x64` if compiling for Win64, or `"Any CPU"` if compiling cross-platform)
69 
70 Make sure that your projects using Colore are also compiled against a matching platform.
71 
72 Razer's SDK installer **will only install the library relevant for your platform**.
73 
74 This means that your apps will need to be compiled twice, once for x86 platforms, and once for x64, unless you are using "Any CPU".
75 They must both be provided when you distribute your application, depending on what platform the user has.
